# my-workout-planner — 화면 정의

> [one-pager](./one-pager.md) 기준. MVP 범위의 화면 목록과 화면 간 흐름을 정의한다.

## 운동 철학 (설계 전제)

> LLM 코치의 모든 개입은 **과학적 근거(점진적 과부하·자가조절)** 를 따른다.

- **효율적인 근력 운동** = 계획된 점진적 과부하를 *추세*로 따르되, 매 세션의 수행(RIR·실제 기록)을 측정해 **다음 계획에 반영**하고, 현장 변수(통증·컨디션·장비)에는 **완주와 안전을 우선해 즉석 조정**하는 것.
- **부하 조정의 비대칭 규칙**:
  - **하향(↓)은 실시간** — 컨디션 난조·통증·어지러움 → 감량/휴식/조기 종료를 즉석에서 적극 권유. 안전·완주 우선.
  - **상향(↑)은 다음 계획으로** — "오늘 잘 된다"는 신호는 신뢰도가 낮고(워밍업·아드레날린), 즉석 증량은 회복·폼을 해친다. 너무 쉬웠다면 그 수행(RIR)을 **기록**해 **다음 계획 생성 시 증량 근거**로 쓴다.
- **측정**: 운동 실행 화면에서 세트별 실제값(무게/횟수)과 **RIR**(남은 반복 수)을 기록한다. 이것이 점진적 과부하 자동화의 데이터 근거.

## 레이아웃 원칙

- **단일 모바일 UI**: 모든 화면을 모바일 기준으로 설계한다. 별도의 데스크톱 레이아웃은 만들지 않는다.
- **데스크톱 표시**: 데스크톱/태블릿에서도 동일한 모바일 UI를 고정 폭(예: 420px) 컨테이너로 화면 중앙에 배치한다. 컨테이너 바깥은 빈 배경.
- **바텀시트 미사용**: 바깥 영역 터치로 실수로 닫히는 것을 막기 위해, 주요 정보·액션은 **독립 화면**으로 둔다.

## 개념 모델

- **루틴(Routine)** = 템플릿. "주 4회 상하체 분할, 근비대" 같은 반복 가능한 운동 설계.
- **계획(Plan)** = 루틴의 **인스턴스**. 특정 날짜에 루틴을 적용해 만든 그날의 구체적 운동. (`루틴 1 : 계획 N`)
- **세션 수행 기록** = 계획을 실행하며 남긴 세트별 실제값·RIR. 다음 계획의 점진적 과부하 근거.

## 화면 목록

| # | 화면 | 목적 | 비고 |
|---|------|------|------|
| S0 | 로그인 | 이메일/OAuth 간단 로그인 | 미인증 시 진입점 |
| S1 | 홈 | 오늘 카드 + 미니 캘린더 + 액션 허브 | 인증 후 기본 화면 |
| S2 | 루틴 생성 (대화) | LLM과 대화하며 운동 루틴 생성 | 구조화 출력으로 루틴 확정 |
| S3 | 루틴 상세 | 확정된 루틴(분할/요일/운동) 확인 | 루틴 단위 보기 |
| S4 | 캘린더 | 월/주 단위로 계획 조망 | 날짜 탭 → 계획 상세 / 빈 날 → 계획 생성 |

> MVP 제외(one-pager): 진척/볼륨 분석, 식단, 소셜/공유, 알림, 다중 루틴 동시 운영.

## 화면별 요약

### S0. 로그인
- **목적**: 사용자를 식별해 루틴/계획/기록을 저장·조회한다.
- **주요 요소**: 앱 소개 한 줄, 이메일 로그인, OAuth 버튼.
- **나가는 흐름**: 로그인 성공 → S1.

### S1. 홈
- **목적**: 오늘 할 운동을 바로 시작하면서, 스케줄을 조망한다. ("오늘" 메인 + 조망 보조)
- **주요 요소**:
  - **오늘 카드** — 오늘 계획 요약(운동 수·상태). 있으면 `운동 시작`, 없으면 `오늘 계획 만들기`.
  - **액션** — `루틴 만들기`(→ S2), `계획 만들기`(→ S5, 날짜 자유).
  - **미니 캘린더** — 주간 미니뷰, 계획 있는 날 표시. 탭 → S6, "전체 보기" → S4.
- **나가는 흐름**: S2 / S5 / S6 / S7 / S4.

### S2. 루틴 생성 (대화)
- **목적**: LLM과 대화로 목표·빈도·분할을 정해 루틴을 만든다.
- **주요 요소**: 대화 메시지 리스트, 입력창, 제안된 루틴 미리보기(구조화 출력), `이 루틴으로 확정`.
- **나가는 흐름**: 확정 → S3.

### S3. 루틴 상세
- **목적**: 확정된 루틴 구조를 확인한다.
- **주요 요소**: 루틴 이름, 분할 방식, 요일별 운동 목록.
- **나가는 흐름**: `이 루틴으로 계획 만들기`(→ S5), 홈으로(→ S1).

### S4. 캘린더
- **목적**: 월/주 단위로 계획을 조망한다.
- **주요 요소**: 월 캘린더, 계획 있는 날 표시(상태별: 예정/완료), 오늘 하이라이트.
- **나가는 흐름**: 계획 있는 날 탭 → S6, 빈 날 탭 → S5(그 날짜로 생성 진입).

### S5. 계획 생성 (대화)
- **목적**: 선택한 날짜에 루틴을 적용해 구체적 계획을 만든다. 이때 LLM이 **과학적 근거 기반 점진적 과부하**를 제안한다.
- **주요 요소**: 대상 날짜, 사용할 루틴 선택, 이전 수행 기록 요약, LLM 제안(예: "지난 벤치 RIR 2 → 2.5kg↑"), `계획 확정`.
- **진입**: `계획 만들기`(홈), 빈 날짜(캘린더), `이 루틴으로 계획 만들기`(S3), `다시 운동하기`(S6 완료 계획).
- **나가는 흐름**: 확정 → S6(또는 바로 S7 시작).

### S6. 계획 상세
- **목적**: 그날 "뭐 하기로 했나 / 뭐 했나"를 본다. **읽기 중심** 화면.
- **주요 요소**:
  - 메타 — 날짜, 원본 루틴, 상태(`예정`/`진행 중`/`완료`).
  - 운동 리스트 — 운동별 목표 세트/횟수/무게.
  - 수행 기록 — 완료 시 세트별 실제값·RIR(점진적 과부하 근거).
- **상태별 주요 액션**:

  | 상태 | 액션 |
  |---|---|
  | 예정 | **운동 시작** (→ S7) |
  | 진행 중 | **이어서 하기** (→ S7) |
  | 완료 | 기록 보기 + **다시 운동하기** (→ S5, 이 계획을 템플릿 삼아 새 날짜 생성) |
- **나가는 흐름**: S7 / S5 / S1.

### S7. 운동 실행
- **목적**: 계획대로 세트를 기록하며 운동을 진행한다. **쓰기 중심** 화면.
- **주요 요소**: 현재 운동, 세트별 기록(무게/횟수 체크 + **RIR 입력**), 다음 운동 이동, `코치에게 물어보기`(→ S8), `운동 종료`.
- **나가는 흐름**: 코치 호출 → S8, 종료 → S6(완료 상태) 또는 S1.

### S8. 운동 중 코치 (대화)
- **목적**: 운동 *중* 발생하는 현장 변수에 LLM 코치가 **과학적 근거로** 대응한다. (대체 운동은 그 한 사례)
- **다루는 상황과 결과** (진행 중 계획에 반영):

  | 상황 | 코치 결과 |
  |---|---|
  | 자리 없음/장비 없음 | **운동 교체** (랫풀다운 → 어시스트 풀업) |
  | 컨디션 난조 | **부하 하향** (무게/횟수/세트 ↓) |
  | 어지러움/통증 | **휴식 권유** 또는 **조기 종료** |
  | 너무 쉬움 | 오늘은 계획 유지, 수행 기록 → **다음 계획에서 증량** 안내 |
- **주요 요소**: 대화 메시지, 입력창, LLM이 제시한 변경안 미리보기, `적용`.
- **나가는 흐름**: 적용/대화 종료 → S7(변경 반영된 세션으로 복귀).

## 핵심 동선 (대표 시나리오 매핑)

one-pager의 대표 시나리오를 화면 흐름으로:

1. 루틴 생성: S1 → **S2**(대화) → S3
2. 전날 계획 요청: S1 `계획 만들기` → **S5**(2026-05-25, 과부하 제안) → S6
3. 헬스장에서 실행 중 대체: S6 → **S7** → **S8**(랫풀다운 자리 없음 → 어시스트 풀업) → S7 → 종료
4. 컨디션 난조 대응: S7 → **S8**("2세트째 어지러움" → 5분 휴식 → 부하 하향/종료) → S7
